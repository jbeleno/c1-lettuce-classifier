# C1 Frontend — Interface Design System

Generated by the `interface-design:interface-design` skill during the initial
build of the React dashboard. Keep this file in sync when adding patterns;
read it before adding new components.

## Direction & Feel

**Lab-grade hydroponic dashboard.** Slate-900 surfaces inherited from the C2
sister project, re-mapped to lettuce growth stages. Single accent per card
— always the predicted-class color. Tabular monospace for every number so
columns read like an instrument readout.

The interface is invisible. The data is what stays in your eye.

## Signature element

**The Growth Spine** — five horizontal dots representing the growth-stage
timeline (`empty_pod → germination → young → pod → Ready`). The active
stage is filled in its class color; the connector lines between reached
stages also fill. Appears in **every route**:

- Predict — full-size next to the result, with the stage token under the active dot
- History — inline (`size="sm"`) on every row
- Metrics — as an axis (`size="axis"`) labelling the confusion-matrix axes
- Models — empty spine inside each model card

If you redesign the spine per route, you've broken the signature. Don't.

Component: `src/components/GrowthSpine.tsx`

## Defaults rejected

| Default | Replacement | Why |
|---|---|---|
| Sidebar with vertical nav | Top bar, underlined active tab | More vertical real estate for data |
| "Drop file here" + cloud icon | **Viewfinder** with four corner brackets | Looks like a microscope / camera frame, fits the lab feel |
| Donut/pie for probabilities | Horizontal monospace bars, only the predicted row in color | Tabular alignment, eye lands on the answer |
| Generic spinner | `Loader2` from Lucide + status text inside the same result card | Loading state lives where the answer will appear |
| Native `<select>` | Native styled with our tokens (only field that survives) | Acceptable here because we control width via CSS |

## Tokens

Lives in `src/index.css` under `@theme`. Tailwind v4 reads CSS variables
directly. **All colors trace back to these primitives** — no random hex.

### Surfaces (whisper-quiet elevation)
- `--color-bg` `#0F172A` — page canvas (slate-900)
- `--color-surface-1` `#1E293B` — default card
- `--color-surface-2` `#334155` — elevated (hover, dropdowns)
- `--color-surface-inset` `#0B1220` — inputs (darker than canvas)

### Borders (low-opacity rgba, never harsh)
- `--color-border-soft`   `rgb(148 163 184 / 0.10)`
- `--color-border`        `rgb(148 163 184 / 0.18)`
- `--color-border-strong` `rgb(148 163 184 / 0.28)`
- `--color-ring-focus`    `#38BDF8` (keyboard focus only)

### Text hierarchy (four tiers)
- `--color-ink`      `#F8FAFC` — headlines, predicted label
- `--color-ink-2`    `#CBD5E1` — body, primary data
- `--color-ink-3`    `#94A3B8` — metadata
- `--color-ink-mute` `#64748B` — placeholders, disabled

### Class accents — the ONLY accents in the app
- `empty_pod`   `#64748B` slate — nothing growing
- `germination` `#A78BFA` violet — early biological
- `young`       `#FBBF24` amber — nascent
- `pod`         `#38BDF8` sky — developing
- `Ready`       `#22C55E` green — harvestable

### Typography
- `--font-sans`: Inter (UI, labels)
- `--font-mono`: JetBrains Mono (every number, every datum)
- `.tabular` utility class — applies `font-variant-numeric: tabular-nums`

## Depth strategy

**Color shifts only**, no shadows. Elevation = +5% lightness of surface.
The card accent (1px top border in the class color) is the only "lift"
signal.

## Spacing

Tailwind defaults (4px base unit). Section gap `gap-6`, intra-card gap
`gap-3`, micro gap `gap-1.5`.

## Border radius

Small — `rounded` (4px) for buttons / chips / inputs.
Card corners default Tailwind `rounded` (4px), no large rounding.

## Animation

- Micro-interactions: 150ms `cubic-bezier(0.2, 0.8, 0.2, 1)`
- Bar fills / page transitions: 240ms with the same easing
- No bounce, no spring
- `@media (prefers-reduced-motion: reduce)` forces all transitions to 0.01ms

## Component patterns (reuse, don't reinvent)

- `Card` — `bg-surface-1`, `border-border-soft`, optional `accent` prop adds
  a 1px top stripe in the class color. Used for every section.
- `ClassChip` — chip with the class color as background-at-18%-alpha, border
  at 35%-alpha, and a 6px filled dot. Sizes `sm | md | lg`.
- `ProbabilityBars` — horizontal bars, only the predicted-class row gets
  its accent fill; the rest are muted.
- `ViewfinderUpload` — drag/paste/click target with corner brackets.
- `WebcamCapture` — vanilla `getUserMedia`, lifecycle-managed.
- `ModelPicker` — themed `<select>` reading from `/api/models`.

## Accessibility hard rules

- Every form input has `htmlFor` matching its id.
- The result region uses `aria-live="polite"` so screen readers announce
  predictions when they land.
- Each probability bar is a `role="meter"` with `aria-valuenow`.
- Icons are Lucide SVGs only — no emojis anywhere.
- Single-color signal is never used alone — labels are always paired with
  text and a chip.
- Focus rings (`:focus-visible`) only show for keyboard, never for mouse
  clicks.

## When in doubt

Run the swap test from the skill: if you could swap the spine for a number
+ icon and the page would feel the same, you've defaulted. Iterate.
